## Whispering v8.0.0: No FFmpeg, Native Recording, and Cleaner Local State

v8 cleans up old paths while Whispering is still pre-launch.

Highlights:
- FFmpeg is gone from the runtime: no install page, recorder backend, decode fallback, or cloud-upload compressor. Decode uses Symphonia plus libopus; uploads use in-app OGG/Opus.
- Desktop manual recording uses CPAL, with a floating overlay for live mic levels, stop, and cancel.
- Whispering can pause system playback while it captures your voice, then resume it afterward. Manual recording pauses for the whole press-to-talk window; VAD pauses per utterance.
- The home capture row now has three clear surfaces: manual recording, voice activation, and uploaded files. Transcripts default to clipboard, with paste-at-cursor as an opt-in behavior.
- Recordings, transcripts, transformations, and completed run history stay workspace-backed. Markdown is a readable export/projection.
- Transformations collapse to pre-replacements, one optional prompt, and post-replacements, with a picker for comparing candidates.
- Provider settings move to `providers.<id>.*`; old device settings are not migrated.
- Local model setup uses recommended defaults and the models folder as the source of truth.

Heads up: bundle ID, timezone, and settings clean breaks may require one device setup pass.

# Whispering v8.0.0: Clean Breaks, No FFmpeg, and a Better Recording Loop

v8 is the release where Whispering stops carrying a lot of old machinery. The desktop app gets a real Epicenter identity, recording gets a visible presence, local audio no longer depends on FFmpeg, and transformations become one understandable prompt flow instead of a pipeline editor.

Because Whispering is still pre-launch, this release also takes the clean breaks now instead of carrying migration code forever.

## Breaking Upgrade Notes

These are the changes to call out before people install v8.

- The Tauri bundle identifier changes from `com.bradenwong.whispering` to `so.epicenter.whispering`.
- The dev bundle identifier is `so.epicenter.whispering.dev`.
- The new app namespace standard is `so.epicenter.<app>`. Future Epicenter desktop apps should follow the same reverse-domain shape.
- The app-data root changes with the bundle identifier. v8 does not migrate old `com.bradenwong.whispering` app data into the new root.
- Device settings do not carry forward from pre-v8 provider-config builds. Users re-enter API keys and device preferences once per device; recordings, transcripts, and transformations live in the workspace store.
- Provider credentials and endpoints are now keyed under `providers.<id>.*`. The old `apiKeys.*`, `apiEndpoints.*`, and `transcription.speaches.*` device-setting keys are no longer read.
- Recordings now require `recordedAtZone`. Older local rows without that field are invalid under the clean-break schema.
- The old per-step Custom base URL is gone. Custom backends now use one global Custom endpoint until named backend profiles land.
- The old transformation step table, step-specific model settings, and step-run history shape are gone.
- FFmpeg is no longer a recording backend, install prerequisite, local decode fallback, or cloud upload compressor inside Whispering.
- Desktop manual recording is CPAL-owned now. The old runtime choice between Browser API, FFmpeg, and CPAL manual recording paths is gone.

## Data, Markdown, and Sync Boundaries

The updater is not meant to delete your recording files. Recordings, transcripts, transformations, and completed run history live in Whispering's workspace-backed app state, while audio files remain device-local artifacts addressed by recording id.

Markdown is still part of the local-first story, but with a clearer role. Generated Markdown is a readable projection/export for inspection, search, copying, and review. The live app state is the workspace, and app data should be changed through Whispering or Epicenter actions rather than by editing generated Markdown.

Transcript text and metadata can move with the workspace. Audio playback still depends on the audio file existing on the device you are using.

## Epicenter App Identity

Whispering now uses the Epicenter desktop app identifier:

```txt
Before: com.bradenwong.whispering
After:  so.epicenter.whispering
```

This is the new pattern for Epicenter desktop apps: `so.epicenter.<app>`. Matter follows the same shape with `so.epicenter.matter`, and new desktop apps should use the same reverse-domain namespace.

This does not change the app name. It is still Whispering. It also does not change the workspace or sync-room identity; those remain separate from the Tauri bundle identifier. The bundle identifier is the OS packaging identity used for app data, single-instance behavior, and platform metadata.

Because platform app-data paths come from the bundle identifier, v8 uses a new app-data root:

```txt
macOS:   ~/Library/Application Support/so.epicenter.whispering/
Windows: %APPDATA%/so.epicenter.whispering/
Linux:   ~/.local/share/so.epicenter.whispering/
```

Older installs used `com.bradenwong.whispering`. If you have local model files or app data from a previous build, keep the old folder until you have confirmed the new build sees what you need.

The app metadata now also names Epicenter as the publisher and copyright owner, matching where the project lives now.

## No More FFmpeg Requirement

FFmpeg has been removed from Whispering's runtime surface. Users should no longer need to install FFmpeg for recording, local transcription, uploaded audio files, Browser API recordings, or desktop cloud-upload compression.

The old path had three separate audio conversion tiers, and the last tier shelled out to a user-installed FFmpeg binary. v8 replaces that with one in-process Rust decode path:

```txt
audio bytes
  -> Symphonia demux and decode for common containers
  -> libopus decode for Opus packets
  -> 16 kHz mono PCM for local engines
```

Desktop recordings still start as WAV because that is the right local interchange format. When a desktop cloud provider needs an upload, Whispering now compresses the WAV to OGG/Opus in-app:

```txt
CPAL recording
  -> WAV on disk
  -> libopus encoder for cloud uploads
  -> audio/ogg sent to the provider
```

That removes the old install page, FFmpeg compatibility warnings, `FfmpegNotFoundError`, FFmpeg compression settings, the desktop FFmpeg RPC/service surface, and the FFmpeg recording backend. The only remaining FFmpeg mentions are historical docs and fixture-generation commands for decoder tests.

## Floating Recording Overlay

Whispering now shows a small floating overlay while recording or listening. It gives you a live microphone meter and lets you stop or cancel without going back to the main window.

On macOS, the overlay is built as a non-activating panel, so clicking stop does not yank focus away from the app you are dictating into. Clicking the body of the overlay is still an intentional way to bring Whispering forward.

## Playback Pauses While You Speak

Whispering can now pause system playback while it captures your voice, then try to resume it afterward. The point is simple: music, podcasts, and videos should not talk over you while you are dictating.

The timing follows the capture mode:

- Manual press-to-talk pauses for the whole press -> release recording window.
- Voice activation pauses only for each detected utterance, not for the whole armed listening session.

VAD resume is debounced, so playback does not flutter between short phrases. The next detected utterance cancels the pending resume; disarming voice activation resumes immediately. On macOS, resume only fires when a CoreAudio output-activity read saw something playing when you paused. Because the macOS Play command is system-wide, it can still occasionally wake a different app than the one you paused.

The feature is off by default, so it never surprises you with playback you did not ask for. Turn it on from the capture-behavior popover on the home pipeline row, or in Recording settings.

## Capture and Delivery Defaults

The home screen now treats capture as three surfaces: Manual, Voice Activation, and Upload File. File import is no longer modeled as a recording trigger; it is a transient surface for dropping audio or video into the same transcription and transformation flow.

Transcript delivery also starts from the safer default: copy to clipboard. Paste-at-cursor is still there, but it is now opt-in because desktop auto-paste needs OS permissions and can surprise people when they are still setting the app up. Whispering now explains the macOS Accessibility upgrade wherever it matters, including auto-paste, without making it sound like a wall before basic recording can work.

## Timezone-Aware Recording History

Recording rows now store both the instant and the display timezone:

```txt
recordedAt:     2026-06-13T16:20:00.000Z
recordedAtZone: America/Los_Angeles
```

That lets the recordings table render a recording in the timezone where it happened, including the zone abbreviation, instead of reinterpreting every old capture in the viewer's current timezone. `updatedAt` stays viewer-local because it is not tied to the original recording location.

## Simpler Transformations

Transformations are no longer an arbitrary multi-step pipeline. Each transformation now has a fixed shape:

```txt
pre-replacements
optional AI prompt
post-replacements
```

That means offline cleanup can happen before and after the model call, while the AI part stays understandable: one prompt, one backend, one model. The old transformation step table, per-step model settings, step editor, and step-run history are gone.

This makes the common cases easier:
- Replace spoken phrases like "new paragraph" before sending text to a model.
- Run one prompt to format, rewrite, summarize, or clean up the text.
- Apply deterministic cleanup after the prompt, such as punctuation or naming fixes.

## Transformations Picker

The old clipboard transform surface has grown into the Transformations picker. Select text in another app, open the picker, choose one or more transformations, and compare candidates before committing one.

The picker supports candidate cards, inline word diffs, multi-transformation selection, and keyboard controls. Ad-hoc candidates stay in memory until you accept one; test runs and abandoned candidates do not clutter your saved run history.

## Provider and Settings Cleanup

Provider setup has one durable shape now:

```txt
providers.openai.apiKey
providers.openai.endpoint
providers.custom.apiKey
providers.custom.endpoint
providers.speaches.modelId
```

This replaces the old split between `apiKeys.*`, `apiEndpoints.*`, and one-off Speaches settings. It also fixes a real bug: OpenAI and Groq transformation completions now honor endpoint overrides the same way transcription already did.

The settings UI follows the same model. Provider credential fields and provider-specific transcription settings now render from provider registries instead of hand-maintained branches. Adding a provider should be a registry change, not a hunt through every settings page.

Fresh setup is also more platform-aware:

- Desktop defaults to the recommended local Parakeet path.
- Web defaults to OpenAI instead of a desktop-only local model.
- Web settings now warn clearly when a synced desktop-local provider cannot run in the browser.
- Cloud and self-hosted transcription adapters use `spokenLanguage` for source-language hints.

## Local Model Setup

Local models are now folder-backed. The models folder under app data is the source of truth, and settings store an entry name instead of an absolute path:

```txt
Before: transcription.whispercpp.modelPath = /Users/you/.../ggml-tiny.bin
After:  transcription.whispercpp.model     = ggml-tiny.bin
```

That avoids duplicating multi-GB models into app data, stops storing machine-specific absolute paths, and lets symlinked models work. The settings screen shows catalog models, custom files dropped into the folder, an Open Models Folder button, and a missing-model notice when the selected entry disappears.

The local model screen now starts from a recommended default for each engine. Download progress is shared between the recommended card and catalog rows, and if the recommended model is already on disk the action becomes Activate instead of Download.

Local models also prewarm when capture starts. Manual recording and armed voice activation can overlap the cold model load with the time you are already speaking, so the transcript has less work left after you stop.

## Faster Local Engines on macOS

Parakeet and Moonshine now force the CPU execution provider on macOS for the int8 ONNX models Whispering ships today. CoreML sounded like the faster path, but these int8 graphs fall back to CPU for the important operators and pay CoreML setup plus partition overhead.

For `parakeet-tdt-0.6b-v3-int8`, the CPU path measured around 3x faster than the previous CoreML auto path. The transcription-complete log now includes `elapsed_ms`, so performance comparisons are easier to reproduce.

## Better Recording Shortcuts and State

Global hotkey capture now uses physical key codes, so shortcuts are less dependent on a US keyboard layout. This helps layouts where the same physical key produces a different character, such as Finnish ISO keyboards.

The default global recording toggle is now `Alt+Space`. Local shortcuts and global shortcuts are separate toggles, manual shortcut entry understands common aliases like `cmd`, `ctrl`, `option`, and `space`, and registration errors are shown instead of being silently swallowed.

Quick press-and-release push-to-talk is also safer. If you release while recording startup is still in flight, Whispering queues the stop instead of latching recording on.

Whispering also stores fewer fake "currently running" states. Transformation runs no longer store a durable `running` result, and recordings no longer store `TRANSCRIBING` as if it were a permanent fact. The process owns liveness; rows store terminal facts. After a crash or interruption, the UI can show retryable interrupted work instead of pretending it is still running forever.

## Startup and Reliability Fixes

Startup is less jumpy too. Whispering waits for workspace settings before running onboarding checks, so persisted local transcription settings should not briefly trigger a false setup prompt.

The home screen now treats transcription setup as an explicit first-run step when the selected provider is not ready, instead of dropping users into the main capture controls with a hidden missing requirement.

Desktop development fails loudly if the web bundle accidentally loads inside Tauri. That catches the stale-dev-server case where the UI looks alive but native capabilities are missing.

Transformation failures are recorded as failed runs instead of getting stuck as running when a provider or step throws unexpectedly.

Bulk transcription now saves the transcripts it creates. Missing local model folders are treated the same way as missing model entries when deleting a model, so cleaning up outside the app no longer turns deletion into a false failure.

## Also in the v8 Window

The last stretch of work also includes repo-wide and adjacent app changes that matter to the v8 story, even if they are not all first-run Whispering features:

- The Epicenter workspace layout now uses the namespace-root shape, with mounts as direct children and `.epicenter/` holding runtime state.
- Mount bootstrap now refuses to adopt a non-empty mount folder before `.epicenter/` exists, protecting user-owned folders from being treated as generated projections.
- The CLI daemon flow is steadier: daemon actions, `/run`, projection teardown, and app-local check commands have been tightened.
- Matter gained a Markdown live preview and optional modeled fields.
- The landing page work in flight reframes epicenter.so around Whispering as the install-tonight product and the managed-folder guarantee.
