# Single-source brand-asset pipeline

- **Status:** Draft
- **Decision:** [ADR 0015](../docs/adr/0015-the-brand-mark-has-one-canonical-source-every-other-form-is-generated.md)
- **Extends:** [ADR 0023](../docs/adr/0023-whispering-separates-its-identity-mark-from-lucide-controls.md)

> Cross-cutting: spans `apps/whispering`, `apps/landing`, and a new shared brand
> module. This is the build plan behind ADR 0015; harvest nothing back here once
> it lands, delete the spec.

## The win in one line

The studio mic exists as six-plus hand-made copies under three names across two
apps with nothing linking them; collapse them to one canonical source plus
generated derivatives, so swapping the mark is the one-file change ADR 0023
already promised.

## Current state (inventory)

The same mark, today, as separate committed files:

| File | Role | Name style |
|------|------|-----------|
| `apps/whispering/src/lib/assets/studio-microphone.png` (160x160) | web hero + sidebar | drawing |
| `apps/whispering/src-tauri/recorder-state-icons/studio_microphone.png` | tray idle (byte-identical) | emoji shortcode |
| `apps/whispering/src-tauri/icons/{icon.png,icon.icns,icon.ico,32x32,128x128,128x128@2x}` | desktop app icon | generic |
| `apps/whispering/static/{favicon.ico,favicon-16x16,favicon-32x32,apple-touch-icon,android-chrome-192,android-chrome-512}` | web favicon + PWA | generic |
| `apps/landing/public/studio-microphone.png` | landing hero + footer | drawing |

Plus the recording-state indicator `recorder-state-icons/red_large_square.png`
(tray recording), also named after its emoji shortcode.

The true source is external (emoji.aranja.com, Apple-style emoji art, unlicensed
per ADR 0023). Every file above is a derivative at an ad-hoc resolution.

**Audit before building:** `apps/landing/src/pages/logo.astro` (a `/logo` route
exists in the build) and `apps/whispering/static/site.webmanifest` icon entries
— fold both into the inventory and the generation outputs.

## Target shape

```
packages/brand/                         (or a brand entry in @epicenter/ui)
  src/whispering-mark.png               canonical source (placeholder today; SVG when owned art exists)
  src/recording-indicator.png          tray recording glyph, role-named
  scripts/generate.ts                   one source -> all raster outputs

apps/whispering/
  src/lib/assets/  ->  import from @epicenter/brand (no local copy)
  src-tauri/icons/*                     generated by `tauri icon <source>`
  static/favicon*, *-chrome-*           generated by scripts/generate.ts
  src-tauri/recorder-state-icons/
    whispering-mark.png                 generated (tray idle)
    whispering-mark-template.png        generated macOS monochrome template (the real divergence ADR 0023 named)
    recording-indicator.png            generated (tray recording)

apps/landing/
  hero + footer  ->  import from @epicenter/brand (delete public/studio-microphone.png)
```

## Waves

1. **Establish the source + module.** Create the brand module, move the
   highest-resolution placeholder in as `whispering-mark.png`, and have both apps
   import it on the web surfaces. Delete `apps/landing/public/studio-microphone.png`
   and the whispering `src/lib/assets` copy in favor of the shared import. No
   generation yet — just one source, imported everywhere the web renders it.
2. **Generate the rasters.** Add `scripts/generate.ts` (sharp or
   pwa-asset-generator) for the favicon/PWA set, and wire `tauri icon` for the
   desktop icons. Document the `source -> outputs` step; commit the outputs.
3. **Tray + role names.** Generate the tray idle/recording/template variants;
   rename `studio_microphone` -> `whispering-mark` and `red_large_square` ->
   `recording-indicator` at the `tauri.tauri.ts` resolver
   (`IDLE`/`RECORDING` paths) and `tauri.conf.json` `bundle.resources`.

## Open questions

1. **Home for the source:** new `packages/brand` vs a `brand/` entry in
   `@epicenter/ui`. `@epicenter/ui` already ships shared UI; a brand entry may be
   the lighter move. Decide against the publish/consume rules.
2. **Commit generated outputs vs gitignore + prebuild hook.** ADR 0015 leans
   commit-with-documented-source; confirm against how Tauri bundling and the
   favicon `<link>`s resolve in CI.
3. **Owned art (art-gated).** Everything above works with the placeholder raster.
   The SVG promotion is a one-file swap and is out of scope until owned art
   exists; ADR 0023 rejected hand-drawn art as amateur.

## Non-goals

- Authoring owned art (deferred, ADR 0023).
- Touching ADR 0023's two-tier identity-vs-control split (it stands).
- The `index.astro` tools-list emoji (a uniform per-product list affordance, not
  the brand mark).

## Verification

- One source file; `grep` finds no second committed copy of the mark.
- `tauri icon` + the favicon script regenerate every raster from the source with
  no manual edits; outputs match what shipped.
- `apps/whispering` and `apps/landing` both typecheck and build; the desktop app
  icon, favicon, in-app hero/sidebar, tray idle, and landing hero/footer all
  render the same mark.
