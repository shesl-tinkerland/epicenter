# SYNC_STATUS (102) Implementation Plan

Wire up the `SYNC_STATUS` (message type 102) round-trip in `@epicenter/sync-client` so the provider tracks whether local changes have been acknowledged by the server.

## Changes

### 1. Add encode/decode helpers — `packages/sync/src/protocol.ts`

- [ ] Add `encodeSyncStatus(localVersion: number): Uint8Array` — encodes `[varuint: 102] [varuint: payload length] [varuint: localVersion]`
- [ ] Add `decodeSyncStatus(data: Uint8Array): { localVersion: number }` — decodes the above wire format
- [ ] Both follow the same pure-function pattern as existing encode/decode helpers

### 2. Export new helpers — `packages/sync/src/index.ts`

- [ ] Add `encodeSyncStatus` and `decodeSyncStatus` to the barrel export

### 3. Add protocol tests — `packages/sync/src/protocol.test.ts`

- [ ] Round-trip test: encode then decode preserves `localVersion`
- [ ] Wire format test: first byte is 102 (`MESSAGE_TYPE.SYNC_STATUS`)
- [ ] Edge case: version 0, large version number

### 4. Add SYNC_STATUS echo to generic server — `packages/sync-server/src/handlers.ts`

- [ ] Add `case MESSAGE_TYPE.SYNC_STATUS:` that returns `{ response: data }` (echo raw bytes back)
- [ ] Matches Cloudflare server behavior at `sync-handlers.ts:220-224`

### 5. Update server test — `packages/sync-server/src/handlers.test.ts`

- [ ] Change test at line 136 from "silently ignores SYNC_STATUS messages (removed feature)" to "echoes SYNC_STATUS messages back to sender"
- [ ] Verify `result.response` equals the input `data` and `result.broadcast` is undefined

### 6. Wire up the client — `packages/sync-client/src/provider.ts`

This is the main change. Sub-tasks:

**a. Version tracking**
- [ ] Add `localVersion = 0` and `ackedVersion = 0` variables, reset to 0 at the start of each `attemptConnection`

**b. Send SYNC_STATUS after local updates**
- [ ] In `handleDocUpdate`, after `send(encodeSyncUpdate(...))`, increment `localVersion` and `send(encodeSyncStatus(localVersion))`
- [ ] Transition `hasLocalChanges` emitter to `true`

**c. Handle incoming SYNC_STATUS echoes**
- [ ] In `ws.onmessage` switch, add `case MESSAGE_TYPE.SYNC_STATUS:` that decodes the echoed version and updates `ackedVersion`
- [ ] If `ackedVersion >= localVersion`, set `hasLocalChanges` to `false`

**d. Replace text ping/pong with SYNC_STATUS heartbeat**
- [ ] Remove `PING_INTERVAL_MS`, `LIVENESS_TIMEOUT_MS`, `LIVENESS_CHECK_INTERVAL_MS` constants
- [ ] Add `IDLE_PROBE_MS = 2_000` and `RESPONSE_TIMEOUT_MS = 3_000`
- [ ] Rework `createLivenessMonitor` internals:
  - `start()`: arm idle timer, reset on every outbound message (`touch` rename or new method)
  - After 2s idle, send `encodeSyncStatus(localVersion)` as probe
  - Arm 3s response timeout — if no message arrives (any message, not just SYNC_STATUS), close socket
  - Only arm the response timeout after at least one SYNC_STATUS echo has been received from the server (track `serverSupportsSyncStatus` flag)
- [ ] Remove text `"ping"` sends from the ping interval
- [ ] Update `handleVisibilityChange` to send SYNC_STATUS probe instead of text `"ping"`
- [ ] Keep the factory function pattern (`start`/`touch`/`stop`)

**e. Expose reactive hasLocalChanges**
- [ ] Create a second `createStatusEmitter<boolean>(false)` for `hasLocalChanges`
- [ ] Set `true` when `localVersion > ackedVersion` after a send
- [ ] Set `false` when echo brings `ackedVersion >= localVersion`
- [ ] Reset to `false` on disconnect (in `disconnect()` method)
- [ ] Clear listeners in `destroy()`

### 7. Update types — `packages/sync-client/src/types.ts`

- [ ] Add `readonly hasLocalChanges: boolean` to `SyncProvider`
- [ ] Add `onLocalChanges(listener: (hasLocalChanges: boolean) => void): () => void` to `SyncProvider`
- [ ] Update JSDoc to mention SYNC_STATUS heartbeat instead of text ping/pong

### 8. Add client tests — `packages/sync-client/src/provider.test.ts`

- [ ] Test initial `hasLocalChanges` is `false`
- [ ] Test `hasLocalChanges` transitions to `true` after local doc update
- [ ] Test `hasLocalChanges` transitions back to `false` after SYNC_STATUS echo
- [ ] Test `onLocalChanges` listener fires on transitions
- [ ] Test `hasLocalChanges` resets to `false` on disconnect
- [ ] Update MockWebSocket `send()` to capture binary SYNC_STATUS frames (currently ignores non-string data already — it captures Uint8Array)

### What NOT to change

- `server-remote-cloudflare/src/sync-handlers.ts` — already works
- `MESSAGE_TYPE.SYNC_STATUS = 102` value
- Browser `online`/`offline`/`visibilitychange` handlers (keep them)
- Supervisor loop architecture / reconnection logic
- SYNC, AWARENESS, QUERY_AWARENESS message handling
- No new dependencies
