# Platform Renderers

Render one source idea into native platform shapes. Change wrappers, not the thesis.

## LinkedIn

Best default:

```txt
short post
  + one strong real visual
  + title overlay when useful
```

Use:

- Personal photo with title overlay.
- Screenshot with title overlay.
- ASCII diagram rendered cleanly.
- Code diff or product screenshot with one claim.
- 5 to 10 lines of text.

Optimize for:

- A specific opinion.
- A real work story.
- A technical lesson.
- A before and after.
- A mistake or changed mind the author can defend.

Avoid:

- Generic long essays.
- Hashtag stacks.
- "Excited to announce" unless the announcement itself is the story.

## X

Best default:

```txt
single sharp post
or 3 to 6 post thread
```

Use:

- Fragments from the source article.
- Screenshot plus one-line claim.
- ASCII diagram.
- Build-in-public note.
- Strong replies to other posts.

Optimize for:

- Frequency.
- Sharp claims.
- Reusable phrasing.
- One idea per post.

## Reddit

Best default:

```txt
subreddit-native post or comment
```

Use:

- Implementation detail.
- Honest tradeoff.
- Concrete code or screenshot.
- A real question to the community.

Rules:

- Rewrite for the subreddit.
- Do not sound like a recycled LinkedIn post.
- Keep self-promotion secondary and only when relevant.
- Prefer comments when the source idea responds to an existing discussion.

## TikTok, Instagram Reels, YouTube Shorts

Best default:

```txt
vertical slide video
  + real photos/screenshots
  + Marp slides
  + voiceover or captions
  + burned-in captions
```

Use the same core video when possible. Change:

- First hook frame.
- Caption.
- Title.
- CTA.
- Pacing if platform feedback requires it.

Video options:

1. Slideshow with real voiceover.
2. Slideshow with captions only.
3. Screen recording with captions.
4. Code or demo clip with voiceover.
5. Face photo hook for the first second, then slides.
6. Product walkthrough with zooms and labels.

## Instagram Static Or Carousel

Best default:

```txt
carousel or photo-backed post
```

Use:

- Personal photo plus lesson.
- Behind-the-scenes work artifact.
- Screenshot sequence.
- Simple visual essay.

Make it more visual and less formal than LinkedIn.

## Medium

Best default:

```txt
500 to 900 word article
  + one thesis
  + one concrete example
  + one to three visuals
```

Optimize for:

- Searchable title.
- Durable explanation.
- Discovery.

## Substack

Best default:

```txt
same article as Medium
  + more personal context
  + continuity with previous thinking
```

Optimize for:

- Relationship.
- Series.
- Reader replies.

## The Ark

Best default:

```txt
publication record
  + destination wrappers
  + attempts
  + performance notes
```

The Ark should own distribution state, not the source idea. Fuji owns finished writing. The Ark owns where it went, what wrapper was used, and what happened next.
